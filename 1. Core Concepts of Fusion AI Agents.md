# 🤖 Core Concepts of Fusion AI Agents

## 📋 Overview

This is a **capability overview** of Oracle Fusion AI Agents — covering what agents are, the types of agents available, how you can build them in **AI Agent Studio**, and the roadmap ahead.

---

## 🧠 What Are AI Agents?

AI agents have **agency** — just like humans, they can:

- **Interpret** user goals
- **Decide** how to accomplish a task
- **Plan** given a high-level objective
- **Take action** and **call tools** — both within Fusion and external systems
- **Operate** within the business context
- **Automate** complex tasks and workflows that were never before possible

> 💡 The ultimate promise of AI agents is **task automation and workflow automation** — solving complex workflows in seconds that previously took hours of manual effort.

---

## 🎯 The Three Types of AI Agents

Agents exist on a spectrum from simple to complex. Oracle Fusion supports all three types:

### ✍️ 1. Authoring Agents (Generate)

Seamlessly integrated into the application experience, these agents leverage **Fusion Data context** to generate content using large language models (Llama, Cohere on OCI, or OpenAI models).

**What they do:**
- Generate summaries, recommendations, or content within a workflow
- Quick request → response interactions embedded in the application

**Examples:**

| Pillar | Use Case |
|--------|----------|
| **HCM** | Generate a first draft of employee goals from a simple prompt — in seconds instead of an hour |
| **Sales/Service** | Summarize a long customer service chat transcript or phone call — seconds instead of 40 minutes |
| **Supply Chain** | Generate a synopsis of a negotiation |

> All shipped **out of the box** in the Fusion AI Agent platform.

---

### 💬 2. Answering Agents (Q&A over Business Context)

These agents take a piece of **business context** — a document, conversation, or knowledge base — and answer questions based on it using **Retrieval Augmented Generation (RAG)**.

**What they do:**
- Upload a document → it gets **vectorized** instantly
- Ask **semantically relevant** questions in natural language (not just keyword search)
- Get deep, context-aware answers

**Examples:**

| Pillar | Use Case |
|--------|----------|
| **HCM** | Benefits Agent — upload a benefits document, compare year-over-year changes, ask about coverage and policies |
| **Sales/Service** | Knowledge Search Agent — query a knowledge base to respond to customer complaints quickly |
| **Supply Chain** | Ask questions over contracts and procurement documents |

> 🔑 These are **RAG-based agents** — capabilities that were never available to enterprise users before generative AI.

---

### ⚡ 3. Action Agents (Reasoning + Tool Use)

The most powerful type — these agents build on authoring and answering capabilities, **plus** they can call tools and take actions within the system.

**What they do:**
- **Decisioning and planning** — complex tasks typically reserved for humans
- **Call tools** — business objects, REST APIs, email, calculators, and more
- **Delegate tasks** to other agents (supervisor → worker hierarchy)
- **Include a human in the loop** for approvals on critical decisions
- Create an **audit trail** of actions and approvals

**Agentic Patterns Supported:**

| Pattern | Availability |
|---------|-------------|
| Hierarchical agents (supervisor + workers) | ✅ Available |
| Single agents with tools | ✅ Available |
| Sequential agents (chained) | 🔜 Coming in Fall |

**Example — New Hire Onboarding Agent:**

A multi-agent team that handles:
- Company policies and culture questions
- Essential resources and supplies ordering
- Facilities information
- Business-related FAQs

> A new employee would spend hours finding these answers — the agent team delivers them in seconds.

---

## 🏗️ AI Agent Studio

Agent Studio is where **all three types of agents** come together. It enables you to:

- **Build agents quickly** using pre-built templates
- **Extend** shipped agents with custom tools and configurations
- **Build custom agents** from scratch
- **Test and validate** agents before deployment
- **Deploy to production** with security and trust built in

### 📦 Pre-Built Templates

Agent Studio ships with templates spanning all agent types (author, answer, action). You can **instantiate** these templates to create your own agents.

### 👥 Agent Teams

Agents work in **teams** with a supervisor-worker hierarchy:

> **Example — Help Desk Supervisor Agent:**
> - Answers payroll questions
> - Explains vacation and benefits policies
> - Communicates policies via email or Slack
>
> The supervisor agent knows which worker agent to delegate each task to — you give it a generic request, and it picks the right agent automatically.

What would have taken several screens and manual steps now gets **automated through a single conversational workflow**.

## 🔧 Agent Tools

Agents access a variety of tools to accomplish tasks:

| Tool | Description |
|------|-------------|
| 📄 **Document Tool** | Upload structured/unstructured documents; perform retrieval, search, and semantic Q&A |
| 📧 **Email Tool** | Compose and send emails (e.g., marketing emails, policy communications) |
| 🔗 **External REST Tool** | Connect to any public REST endpoint with parametric examples for the LLM |
| 🧮 **Calculator Tool** | Perform computations as part of a task |
| 📊 **Business Object Tool** | Access any Fusion business object via OpenAPI — supports POST, PATCH, DELETE operations with slot filling |
| 👤 **Human-in-the-Loop** | Pause for human approval on critical actions; creates an audit trail |

### 🔒 Data Security — Built In, Not Bolted On

- All tools respect **role-based access controls**
- **Enterprise data protection and privacy** are native to the platform
- Data integrity stays intact across all operations
- Security boundaries are enforced within each application context

---

## 📄 System Prompts

Every agent starts with a **system prompt** that defines:

- **Persona** — who the agent is
- **Scope** — what it can and cannot do
- **Tools** — which tools it can access and what each tool accomplishes
- **Boundaries and context** — the limits of the agent's operation
- **Multi-language support** — works across a variety of languages

---

## 👤 Human-in-the-Loop

A natively built-in capability for tasks that require human oversight:

- Agents **pause** for human review before critical actions
- Humans **collaborate** with agents and give consent
- Creates an **audit trail** documenting who approved what
- Gives humans **control** over autonomous agent decisions

---

## ✅ Quality Evaluation & Testing

### 🧪 Pre-Launch Testing

- Test agents with **numerous example queries** before production
- Compare **expected vs. actual responses**
- Measure: **correctness**, **latency**, **safety parameters**
- Both **automated LLM-based grading** and **human scoring** are supported

### 📈 Production Monitoring — Metrics Dashboard

- Track agent **performance** and **quality adherence**
- Monitor **query resolution time**
- Capture **every single query** for review
- Identify agents that are taking too long or operating outside parameters

### 🔍 Agent Tracing

A **step-by-step walkthrough** of what an agent does on each query:

- Understand the agent's reasoning path
- Use as a **debugging tool** — find where an agent went off track
- Fine-grained, granular visibility into agent behavior

## 💰 Embedded AI vs. Custom AI

### 🆓 Embedded AI (Included with Fusion Apps)

- **Hundreds** of pre-built agent features and templates
- Powered by LLMs hosted in OCI (Llama, Cohere, etc.)
- Modify prompts, use guardrails, and all built-in capabilities
- **Included at no additional charge**

### 🔧 Custom AI (Advanced Tier)

For when out-of-the-box agents don't fully meet your needs:

| Capability | Details |
|-----------|---------|
| Build custom agent teams from scratch | Full flexibility to create and launch your own agents |
| Extend out-of-the-box agents | Add new tools, external REST services, MCP server connections |
| Advanced LLMs | OpenAI GPT-4.1 and similar models with higher token limits |
| Bring Your Own LLM | Use any LLM of your choice — regional, specialized, or task-specific |
| Multi-modality | Image generation, voice-based agents, chart/image Q&A |

### 💲 Pricing

- **User/employee-based agent seats** — fixed price per month
- Designed for **predictability and simplicity**
- Unlike consumption-based models, you have **clear control** over spend

---

## 🧭 Key Takeaways

1. **Three types of agents** — Authoring (generate), Answering (RAG-based Q&A), and Action (reasoning + tool use) — all assembled in Agent Studio.
2. **Pre-built templates** ship out of the box across HCM, SCM, Sales, Service, and more — extensible and customizable.
3. **Agent teams** use a supervisor-worker pattern to automate complex, multi-step workflows.
4. **Security is built in** — role-based access, data protection, and privacy are native, not bolted on.
5. **Human-in-the-loop** ensures oversight on critical decisions with a full audit trail.
6. **Quality tools** let you test, evaluate, trace, and monitor agents before and after production launch.
7. **Embedded AI is free** with Fusion apps; **Custom AI** unlocks advanced models, BYOLLM, and multi-modality.
